admin_postinstall_eaccelerator_condition (line 27)

Checks if the eAccelerator caching method is enabled. This check should be done through the 3.x series as the issue impacts migrated sites which will most often come from the previous LTS release (2.5). Remove for version 4 or when eAccelerator support is added.

This check returns true when the eAccelerator caching method is user, meaning that the message concerning it should be displayed.
